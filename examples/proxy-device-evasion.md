# 代理 IP 与设备对抗 / Proxy & Device Evasion

## Input

```text
某批请求的 IP 分布很分散，但设备指纹高度相似。
部分请求疑似来自模拟器或云手机。
这些账号行为集中在登录、领券、抢购几个接口。
请帮我分析风险。
```

## Expected Output Shape

### 风险判断

网络环境伪装 + 设备环境对抗 + 自动化业务操作组合风险。攻击者可能通过代理 IP 分散来源，同时复用设备环境或伪造设备参数。

### BREAK 映射

- `R0099` 黑 IP 识别绕过
- `R0050` 风险设备识别绕过
- `R0001` 流程自动化
- `R0034` 自动化养号
- `R0003` 恶意抢购

### 黑灰产链路假设

```text
代理 IP 池 → 云手机 / 模拟器 / 群控 → 批量账号登录 → 自动化领券 / 抢购 → 权益套利或商品转卖
```

### 观测信号

- IP 分散但设备指纹相似
- 设备参数组合异常
- 传感器数据缺失
- UA、分辨率、系统版本模板化
- 请求集中命中高价值接口
- 单设备或设备族关联多个账号
- 登录、领券、抢购行为链路过短

### 建议动作

- 建立设备族识别
- 对 IP 风险和设备风险联合建模
- 对高价值接口增加行为序列校验
- 对疑似云手机/模拟器请求降权或复核
- 对账号-设备-IP 三元组做聚类监控
