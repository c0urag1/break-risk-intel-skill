# 商家刷单与虚假评价 / Fake Transaction

## Input

```text
多个商家的订单来自同一批账号和相似地址。
这些订单评价内容高度相似，物流轨迹也比较异常。
部分商品短时间内销量和好评突然上涨。
帮我分析可能的商家作弊风险。
```

## Expected Output Shape

### 风险判断

商家侧虚假交易 / 虚假评价。风险可能来自商家、刷单团伙、账号资源方之间的协作。

### BREAK 映射

- `R0017` 虚假交易
- `R0056` 虚假评价
- `R0004` 虚假发货
- `R0063` 重复铺货
- `R0033` 僵尸店铺

### 黑灰产链路假设

```text
商家 → 购买刷单服务 → 号商提供账号 → 刷手/脚本下单 → 空包或异常物流 → 批量好评 → 提升搜索排序和转化率
```

### 观测信号

- 多账号共享地址、设备、支付工具
- 多个订单集中流向同一商家
- 评价文本相似
- 评价时间与收货时间不合理
- 物流轨迹异常或重复
- 商品销量突增但真实转化不足
- 退款、售后、投诉与销量不匹配

### 建议动作

- 建立商家-账号-地址-物流单号图谱
- 检测评价文本相似度
- 对异常物流轨迹做规则识别
- 对销量突增商品做分层复核
- 对商家实施阶梯处罚和权益限制
