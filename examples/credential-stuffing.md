# 撞库与账号接管

案例类型：模拟案例。
来源：分析框架总结 + BREAK 风险 ID。

## 输入

```text
最近登录接口失败率明显升高。
很多账号在短时间内从不同 IP 尝试登录。
部分成功登录后立即修改手机号、绑定新设备、使用账户余额。
帮我判断这是不是撞库或账号接管。
```

## 已知事实

- 登录接口失败率明显升高。
- 多账号短时间内从不同 IP 尝试登录。
- 部分成功登录后立即修改手机号、绑定新设备、使用账户余额。

## 合理假设

| 假设 | 置信度 | 依据 | 证据缺口 |
|---|---|---|---|
| 存在撞库攻击 | 高 | 登录失败率升高、多 IP 多账号尝试 | 需要账号/IP/密码错误类型/登录速率验证 |
| 部分账号发生接管 | 高 | 成功后立即修改绑定和消耗资产 | 需要用户历史设备、登录地、敏感操作确认 |
| 代理 IP 或自动化资源参与 | 中 | 多 IP 短时间尝试 | 需要 IP 风险、ASN、设备指纹、请求节奏 |

## 攻击标的

- 用户账号登录入口。
- 账号内余额、积分、优惠券等资产。
- 绑定手机号、设备和敏感操作链路。

## 角色分工

| 角色 | 状态 | 置信度 | 依据 | 证据缺口 |
|---|---|---|---|---|
| 凭证资源持有方 | 推断 | 中 | 多账号登录尝试 | 无法直接证明密码库来源，需要错误类型和命中率 |
| 自动化/工具方 | 推断 | 中 | 短时间批量尝试 | 需要请求节奏和客户端指纹 |
| 代理/IP 资源方 | 推断 | 中 | 多 IP 尝试 | 需要 IP 风险标签和 ASN 聚类 |
| Cash-out / asset consumer | 已观察 / 推断 | 中 | 成功后使用余额 | 需要资金流和资产去向 |

## 资源偏好

- 泄露凭证或密码组合：用于批量验证，不能描述获取或验证方法。
- 代理 IP：用于分散请求和绕过频控。
- 设备环境：用于模拟登录环境。
- 资产消耗路径：用于余额、积分、券等变现。

## 概念级作弊链路

```text
凭证资源 → 批量登录尝试 → 命中有效账号 → 修改绑定/设备 → 消耗余额/积分/券 → 资产转移或转卖
```

## 变现路径

- 使用账号余额、积分或优惠券。
- 转移虚拟资产。
- 转卖被接管账号或账户权益。

## BREAK 映射

- `R0032` 撞库
- `R0032-001` 撞库攻击
- `R0084` 账号盗用
- `R0090` 账号接管
- `R0088` 自动化登录风险
- `R0099` 黑 IP 识别绕过，若存在代理/IP 对抗证据

## 观测信号

- 登录： 失败率、错误类型、账号/IP 交叉尝试、成功率。
- 账号： 登录地变化、历史设备差异、敏感操作时间间隔。
- 设备/IP： 新设备绑定、IP/ASN 聚类、代理标签。
- 资产： 余额/积分/券消耗、交易对象、转移路径。
- 用户保护： 用户申诉、找回、通知响应。

## 风控切点

- 异常登录二次验证。
- 成功登录后的敏感操作冷却期。
- 新设备资产操作限制。
- 高风险会话限制余额/积分/券使用。
- 用户通知、冻结保护和找回流程。
- IP、设备、账号三维频控。

## 误伤风险

- 用户换设备、旅行、网络运营商切换可能导致登录异常。
- 忘记密码或活动登录高峰也可能增加失败率。
- 应结合登录失败模式、成功后的敏感操作和资产消耗判断。
